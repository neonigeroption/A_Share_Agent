## Slide 2: 行业痛点与课题价值 (为什么要做这个？)
- **痛点一：大模型存在“幻觉”与时效性黑洞**
  （传统的 LLM 无法处理今天才发生的暴跌，会依靠历史参数胡编乱造）
- **痛点二：散户主观情绪干扰大**
  （需要一个绝对无情、只看数据与利空逻辑的交易机器）
- **技术破局**：
  引入 **FAISS 局部计算** + **Memory Caching 防爬屏蔽** + **LLM Agent 反向推演**。

*🎤 讲稿：“当前 AI 领域有一大痛点：大炮打不了蚊子。也就是参数极高的模型，却没办法处理毫秒级突变的金融短线行情，甚至会用错误的过时数据严重误导交易。因此，我没有局限于只做一个壳子去调用 ChatGPT，而是打造了一条涵盖底层网络爬虫、向量数学模型、自动化推演反馈的全生命周期管线。”*

## Slide 4: 核心护城河 (一) —— 向量 RAG 的设计逻辑
- **机制概述**：避免无效长文本填充 -> 进行 Overlap (重叠区) 切片。
- **数学本质**：千维向量的 `IndexFlatL2` 欧氏距离计算。
- **项目收益**：过滤了 90% 的无效信息，极大地节约了 Token 并提升响应速度。

*🎤 讲稿：“我在系统里运用了目前最成熟的 RAG 思维：不求用算力硬推全本，而是用数学找局部。比如系统在判断‘贵州茅台为何大跌’时，不是把上万字的新闻全部发给 GPT，而是通过转码算距离，只提取三句话，这保证了每一条逻辑都是精确无误的实证分析。”*

---

## Slide 5: 核心护城河 (二) —— 主动型 ReAct Agent
- **与传统 AI 的区别**：它自己有手脚。
- **工作机制：Function Calling**
  `User (提问)` ➡️ `LLM (我要调函数)` ➡️ `Local (拦截执行抓取代码)` ➡️ `LLM (收到执行结果)` ➡️ `Final (分析出研报)`

*🎤 讲稿：“本项目之所以叫 Agent 而不是 Chatbot，就在于自动化。各位稍后可以演示中看到，我在聊天框输入股票后，不是通过按钮触发的搜索，而是 AI 解析了我的自然语言意图，反向命令后端的 Python 脚本去底层数据接口扒了一组最新报表给我。”*
