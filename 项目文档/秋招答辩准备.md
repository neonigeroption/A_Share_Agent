## 🎯 核心一：Streamlit 状态驻留机制 (`app.py`)

在 `app.py` 中，你拥有了一个能在简历上大吹特吹的亮点：**“攻克了响应式 Web 框架的高频重载陷阱”**。

**关键代码行位置**：`app.py` 内部关于“执行深度扫描”以及“模拟买入”的按钮处。
```python
if st.button("🔍 执行深度扫描", type="primary"):
    ...
    # 将搜索目标存入 session_state，防止内部按钮由于嵌套限制失效
    st.session_state.target_keyword = search_keyword
```
**你要学懂的逻辑：**
传统 Web 框架（比如 Vue/React）按钮点击只会局部刷新。但 Streamlit 是 **自上而下完整重构运行（Top-down execution）** 的。
如果我不把 `target_keyword` 存进系统自带的字典 `st.session_state` 里，当用户点击下一步的“买入”按钮时，整个 Python 脚本从第 1 行重新执行。此时“执行扫描”这个动作被判定为 False，原本在里层的“买入”代码连看都不会被多看一眼，操作当场消失！
**面试官杀手锏：** “你能解释下 Streamlit 的 Session State 生命周期吗？”
**你的回答：** “在处理表单联动的场景下，我会利用 `on_click` 回调注入函数（`handle_buy_click`），并且主动接管核心变量寄存到 `session_state`。这样无论组件树怎么刷新重绘，逻辑状态流都不会断裂。”

## 🎯 核心二：向量空间的降维打击 (`vector_rag.py`)

RAG 是如今 AI 极具含金量的热词，你要清楚数据在里面的完整流转链路。

**关键代码行位置**：`vector_rag.py` 里面的 `build_faiss_index` 与 `search_relevant_chunks` 函数。
```python
def chunk_text(text, chunk_size=200, overlap=50): ... # 重叠切片
# ---
vectors = get_embeddings_batch(chunks)  # 调用智谱转成 2048维浮点
index = faiss.IndexFlatL2(dimension)    # L2 欧氏距离暴力检索
distances, indices = index.search(query_vector, top_k)
```
**你要学懂的逻辑：**
1. **Overlap (重叠区)**：为什么要设置 `overlap=50`？如果一句重要的话是“贵州茅台/马上涨停”，刚好前一个切块切到了“贵州茅台”，下一个切块从“马上涨停”开始。这句语意就被切断了，搜不出了！Overlap 机制能保证前后块有 50 字重复，守住语义连贯。
2. **FAISS 选型**：为什么选 `IndexFlatL2`？因为我们单次个股新闻几万字也就是几十个 Block 块，采用**L2（欧几里得几何距离）穷举搜索效率已经极高**，没必要引入 HNSW 等近似临近图算法徒增复杂度。

## 🎯 核心三：反爬与内存屏障 (`market_tool.py`)

**关键代码行位置**：`market_tool.py` 中所有的 `@st.cache_data(ttl=1800)`
```python
@st.cache_data(ttl=1800)
def fetch_all_market_data():
    return ak.stock_zh_a_spot_em()
```
**你要学懂的逻辑：**
你每一次切页面，大模型每一次查数据，都会触发这行代码。如果没有上面那行 `@st.cache_data` 注解装饰器，你的 IP 五分钟内就会被东方财富拉黑。
**底层原理**：这是 AOP（面向切面编程）的缓存思想。Streamlit 会对函数参数做 Hash。第一次执行后，它把拉回来的几千条数据（DataFrame）塞进服务器内存。在接下来的 1800 秒（半小时）内，所有同样的查询指令，都直接穿透底层 API，瞬间把内存里的数据喂给前端页面。

## 🎯 核心四：让 AI 长出手脚 (`agent_tools.py`)

Function Calling 是大厂对 AI 的最高要求之一，证明你具备驾驭多模态 Agent 的能力。

**关键代码行位置**：`TOOL_DEFINITIONS` 的 JSON Schema 定义和 `run_agent()`。
```python
"type": "function",
"function": {
    "name": "get_single_stock_quote", # 工具名
    "description": "获取个股的实时行情数据...",
}
```
**你要学懂的逻辑：**
当你命令 AI 查茅台时：
1. **第一次发请求**：你把这句话和上面这个描述工具的 JSON 字典一起发给 GLM-4。
2. **AI 的响应**：它并没有回答纯文本（因为它没联网），而是返回了一段 `finish_reason="tool_calls"` 的机器报文，告诉你“我要调用刚才那个查股票的函数，参数是‘贵州茅台’”。
3. **本地接管拦截**：我们的 `try-except` 或逻辑判断拦截到了它，你在本地执行 `get_single_stock_quote("贵州茅台")`。
4. **第二次发请求**：你把刚爬回来的价格“1800元，涨幅1%”打包，告诉 AI ：“老外，这是执行结果”。
5. **最终推演**：AI 基于拿到的数据，输出最后那段精彩的涨跌分析。

你能把上面这些内容转化为自己的口语表达，这个项目你就能拿满分！
