# A_Share_Agent 系统架构与工作流程图鉴

本指南将深入剖析 `A_Share_Agent` 量化决策终端的工程结构。我们将从宏观架构图开始，然后逐一拆解每个核心组件 `.py` 文件的具体职能，最后串联出系统的真实运行生命周期。

---

## 1. 系统宏观架构 (The Four-Layer Architecture)

本项目遵循高内聚、低耦合的模块化设计，宏观上分为四层结构：

```text
+-------------------------------------------------------------------------+
|                  4. 前端交互与沙盒博弈层 (User/UI Layer)                 |
|   [app.py] (Streamlit WebApp)  |  [portfolio.py] (模拟持仓状态管理)        |
+-------------------------------------------------------------------------+
                                   ▲
                                   | (Prompt + Session 拦截)
                                   ▼
+-------------------------------------------------------------------------+
|                  3. 大脑决策推演层 (ReAct Agent Layer)                   |
|   [agent_tools.py] / [agent_brain.py]   (Function Calling 循环调度)       |
+-------------------------------------------------------------------------+
                  ▲                                      ▲
                  |                                      |
                  ▼                                      ▼
+-----------------------------------+  +----------------------------------+
| 1. 近端记忆层 (Vector RAG Layer)    |  | 2. 实时感知层 (Market Data Layer)  |
| [vector_rag.py] & [rag_search.py] |  | [market_tool.py]                 |
| (FAISS + 智谱 Embedding)           |  | (AKShare 数据源 + Memory Cache)   |
+-----------------------------------+  +----------------------------------+
```

---

## 2. 核心模块与代码级职能拆解 

### `app.py`（系统大动脉 / UI 总控）
- **具体做什么**：这是整个 Streamlit 应用的入口文件。负责渲染页面布局（分栏配置）、画图（调用 Plotly 绘制势能仪表盘和柱状图、K线图）。
- **特殊职责**：维护生命周期。它承担了 `st.session_state` 的全局状态保持，以解决表单刷新嵌套丢失的经典 Bug。负责将用户的点击动作（例如点击“执行深度扫描”或“模拟买入”）映射到具体的底层功能回调函数上。

### `market_tool.py`（外挂探针 / 数据源）
- **具体做什么**：这是整个应用的“爬虫感知系统”，底层全部基于 `AKShare` 接口。
- **特殊职责**：实现了高级的缓存控制。通过内置的 `@st.cache_data(ttl=1800)` 装饰器构成内存级防封禁装甲。无论是抓取顶部个股、底部爆仓股、还是 60 日历史 K 线指标（MA/MACD），都在这里汇算并转换为标准的 Pandas DataFrame 格式向外抛出。

### `rag_search.py`（资料搜集兵）
- **具体做什么**：专门负责“脏活累活”，去东方财富网底层抓取未加密的实时舆情与新闻公告。
- **特殊职责**：为了应对断网或者接口阻断，内部封装了正则表达式净化提取处理以及极限重试策略，将 HTML 富文本洗净为纯文本。

### `vector_rag.py`（记忆海马体 / 语义分析核心）
- **具体做什么**：将 `rag_search.py` 抓回来的大段新闻转化为可以精准数学计算的“高维语义空间”。
- **特殊职责**：
  1. 调用 `chunk_text` 将几万字新闻安全切片。
  2. 调用 OpenAI (智谱) `embedding-3` 把文本变成 NumPy 数组矩阵。
  3. 调用 `FAISS` 进行 L2 距离的暴力检索。
  4. 为系统提供极为纯净、相关性极高的“局部证据片段”，防止大模型产生胡说八道的“幻觉”。

### `agent_tools.py` 与 `agent_brain.py`（决策司令部）
- **具体做什么**：通过自然语言，让大模型自己决定下一步该执行什么代码。
- **特殊职责**：这是一个 **ReAct (Reasoning and Acting)** 发动机。在 `agent_tools.py` 内定义了标准的 JSON Schema（如 `get_single_stock_quote`），当收到外部指令时，它开启自动挡，自己决定调用工具查行情 → 反馈给模型 → 自己再决定查舆情片段 → 最后整合撰写并返回最终报告。

### `portfolio.py`（沙盒清算所 / 持久化系统）
- **具体做什么**：承载“模拟持仓”功能。这是一个带有文件状态 IO 的业务逻辑模块。
- **特殊职责**：自动记录每一次买入时的成本均价平滑计算。在读取状态时，动态去 `market_tool.py` 抓取现价，以此算出“现价、持有市值、浮动盈亏（百分比/金额）”，供 前端 `app.py` 画出饼图和盈亏看板。

## 3. 标准工作流程剖析 (Workflow)

为了理解多文件如何联动，我们可以演练一个标准的用户交互生命周期：**“用户在输入框打出‘贵州茅台’，希望能查询情况并最终买入”**。

### 阶段一：触发与数据拉取 (The Trigger)
1. 用户在前端页面 `app.py` 输入“贵州茅台”并回车。
2. `app.py` 触发，调用 `market_tool.get_single_stock_quote("贵州茅台")`。
3. 如果 30 分钟内查过，直接从本地内存闪回数据；如果没有，`market_tool` 向底层接口发起请求，拉回它的最新净值、涨跌幅、成交量，返回 DataFrame。

### 阶段二：增强认知捕获 (The RAG Execution)
1. `app.py` 中的公共流水线模块此时调用 `vector_rag.rag_retrieve()`。
2. 该函数向下透传，驱使 `rag_search` 拉取最新的数篇东方财富新闻。
3. `vector_rag` 把新闻瞬间切完并调用 `embedding API`，同时把目标意图（“这只股票最近怎么了”）也变成 Vector，在 FAISS 里做最近邻搜索计算，拿到最相关的 5 句核心结论。

### 阶段三：大脑深度推演 (The Brain Inference)
1. `app.py` 带着 **[阶段一的实时盘面数字]** + **[阶段二的舆情摘要文本]** 这两组弹药，拼接游资专用指令 Prompt。
2. 呼叫大模型 `client.chat.completions.create(stream=True)`。
3. 大语言模型化身无情的交易员，将逻辑从流接口吐出，Streamlit 同步把这段文字打字机般呈现在网页上。

### 阶段四：清算与持久化 (The Settlement)
1. 听取了 AI 的分析，用户决定入场，填入买入 “1000 股”，点击 “✅ 模拟买入”。
2. Streamlit 触发回调，`portfolio.add_position()` 被激活。
3. `portfolio` 将 1000 股乘以刚刚查到的现价算出总金额，跟历史买入成本合并（均价法），并将结果持久化保存到 `portfolio_data.json` 文件中。
4. 页面利用修改过的 Session State 重载，重新扫描持仓 `json` 并抓取现价，底部盈亏面板瞬间刷新，完成整个应用环路。
