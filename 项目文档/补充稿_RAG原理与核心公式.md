## 补充 Slide 1：什么是 RAG？（概念引入）

**标题**：RAG — 检索增强生成（Retrieval-Augmented Generation）

**核心观点（3 条 bullet）**：
- **痛点**：大模型只知道训练截止日之前的知识 → 问它"今天贵州茅台为什么暴跌？"它会**编造**（幻觉问题）
- **解法**：先**检索**（Retrieve），再**增强**（Augment），最后**生成**（Generate）
  — 不让 LLM 凭空回答，而是先帮它找到最新的证据，再让它基于证据推理
- **本项目 RAG 管线**：
  ```
  新闻抓取 → 文本分块 → 向量编码 → FAISS 索引 → 语义检索 → 注入 Prompt → LLM 推理
  ```

**配图建议**：画一个对比图 —— 左边是"无 RAG 的 LLM"（输入问题 → 输出幻觉），右边是"有 RAG 的 LLM"（输入问题 → 检索证据 → 输出有据分析）。

*🎤 讲稿："原始 PPT 里我提到了向量 RAG，但没有展开讲。这里我补充一下核心原理。RAG 的全称是 Retrieval-Augmented Generation，检索增强生成。简单说，就是在让大模型回答之前，先用数学方法帮它找到最相关的实时信息，这样它的回答就不是凭空编造，而是有据可依。"*

## 补充 Slide 2：文本分块与向量编码（Chunking + Embedding）

**标题**：Step 1-2 — 文本切块与语义向量化

### 文本分块（Chunking）

**问题**：一篇新闻几千字，不可能全塞给 LLM → 需要切片

**本项目参数**：
- 块大小 `chunk_size = 200` 字符
- 重叠窗口 `overlap = 50` 字符（防止关键信息被截断）

**示意图**：
```
原文: [████████████████████████████████████████████████]
块1:  [████████████]
块2:       [████████████]      ← overlap 50字
块3:            [████████████]
```

### Embedding（文本 → 向量）

**核心思想**：把人类文字映射成一个 **2048 维浮点数组**

**数学表达**：
```
E: Text → ℝ²⁰⁴⁸

例：
E("股价暴涨涨停封板") = [0.12, -0.03, 0.45, ..., 0.08]   ← 2048个数字
E("天气晴朗适合出游") = [-0.31, 0.72, -0.15, ..., 0.44]  ← 2048个数字
```

**关键性质**：语义相近的文本 → 向量空间中距离更近

**本项目实现**：调用智谱 `embedding-3` 模型，一次 API 调用即可将多段文本批量编码

*🎤 讲稿："第一步是分块。一篇新闻可能有几千字，但跟我的问题相关的可能只有两三句话。所以我把新闻按 200 字切开，并且相邻块重叠 50 字，防止关键信息被切断。第二步是编码。通过调用智谱的 embedding-3 模型，每一段文字被转化成一个 2048 维的浮点数组。这个数组的数学性质是：语义越接近的文本，它们的向量在空间中距离越近。"*

## 补充 Slide 3：FAISS 向量检索与核心公式（Indexing + Search）

**标题**：Step 3-4 — FAISS 向量索引与语义检索

### 核心公式 ①：L2 欧氏距离（本项目使用）

$$d(\\vec{a}, \\vec{b}) = \\sqrt{\\sum_{i=1}^{n} (a_i - b_i)^2}$$

其中 $n = 2048$（向量维度）

**直觉理解**：
- 距离越小 → 语义越相关
- $d = 0$ → 完全相同的文本

### 核心公式 ②：余弦相似度（业界常用对比）

$$\\text{cos\\_sim}(\\vec{a}, \\vec{b}) = \\frac{\\vec{a} \\cdot \\vec{b}}{\\|\\vec{a}\\| \\times \\|\\vec{b}\\|}$$

- 值域 $[-1, 1]$，越接近 $1$ 越相似
- L2 距离与余弦相似度在归一化向量上等价

### FAISS 检索过程

```
① 查询文本 → Embedding → 查询向量 q ∈ ℝ²⁰⁴⁸
② 在索引中计算 q 与所有文本块向量的 L2 距离
③ 取距离最小的 Top-K 个块（本项目 K=5）
④ 返回最相关的 5 段原始文本
```

**本项目配置**：
| 参数 | 值 | 含义 |
|------|----|------|
| 索引类型 | `IndexFlatL2` | 暴力搜索（精确但慢，适合小数据量） |
| 向量维度 | 2048 | embedding-3 输出维度 |
| Top-K | 5 | 返回最相关的 5 个文本块 |
| 数据规模 | ~50 个块/次 | 10 条新闻 × ~5 块/新闻 |

*🎤 讲稿："这是整个系统最核心的数学。当用户问'贵州茅台为什么跌了'，系统先把这句话也变成一个 2048 维向量，然后在 FAISS 索引里计算它和每一个新闻块之间的欧氏距离。公式就是这个 L2 距离公式——对 2048 个维度的差值求平方和再开根号。距离最小的 5 个块，就是跟用户问题语义最相关的 5 段新闻。值得注意的是，我选用 IndexFlatL2 而不是近似索引，因为我们每次只有几十个块，暴力搜索完全够用，而且结果是精确的。"*

## 补充 Slide 4：RAG → Prompt → LLM 推理（Augment + Generate）

**标题**：Step 5-6 — 检索增强 + 大模型推理

### Prompt 注入策略

```
┌─────────────────────────────────────────────┐
│  System Prompt                              │
│  "你是一个言辞犀利的顶级游资..."              │
├─────────────────────────────────────────────┤
│  User Prompt                                │
│                                             │
│  【量价数据】(来自 market_tool)              │
│   代码    名称      最新价   涨跌幅   成交额  │
│   600519  贵州茅台   1800    -3.2%   50亿    │
│                                             │
│  【RAG 检索命中】(来自 FAISS Top-5)         │
│   [片段1] 距离0.32: "茅台因消费税传闻..."    │
│   [片段2] 距离0.45: "北向资金今日净流出..."   │
│   [片段3] 距离0.51: "白酒板块整体承压..."     │
│                                             │
│  请分析：主力意图、风险点、止损位建议          │
└─────────────────────────────────────────────┘
```

### 为什么 RAG 比"全文灌入"更优？

| 维度 | 全文灌入 | RAG 精准检索 |
|------|----------|-------------|
| Token 消耗 | ~8000 token | ~1500 token（节省 80%） |
| 幻觉风险 | 高（信息过载导致迷失） | 低（只看最相关的证据） |
| 响应速度 | 慢（处理大量无关文本） | 快 |
| 精准度 | 低（大海捞针） | 高（数学筛选后的精华） |

*🎤 讲稿："最后一步就是把检索到的精华片段，连同实时行情数据一起注入到大模型的 Prompt 里。相比把整篇新闻全部灌进去，RAG 检索后的 Prompt 只有原来的五分之一大小，但信息密度反而更高。这样大模型的推理就不是凭空编造，而是基于刚刚发生的真实事件做分析——这就是 RAG 消除幻觉的核心机制。"*
