# A_Share_Agent 全栈量化决策终端 —— 产品报告

## 一、 项目背景与痛点分析

在当前的金融决策与 AIGC 融合赛道中，目前绝大多数系统仍停留在**“静态对话机器人”**的初级阶段。对于 A 股超短线交易而言，传统的 LLM 存在以下不可忽视的痛点：
1. **严重的信息滞后与幻觉**：通用大模型无法获取实时的盘口异动数据与突发利空/利好舆情。
2. **缺乏多维叠加分析能力**：面对复杂的量化行情（K线、成交量、资金对倒流向），纯文本大模型无法处理冰冷的数字。
3. **高频防爬墙**：免费的公共金融网（新浪财经、东方财富）存在严苛的防反爬措施，常规并发采集极易被封锁源 IP。

本产品 `A_Share_Agent` 应运而生。它定位为**一个结合实时数据流、本地化向量认知库 (FAISS) 与 ReAct Function Calling 机制的全自动化量化侦察兵**，专门为短线高频博弈提供逻辑推演保护。

## 二、 目标受众特征
- **游资风格交易员**：不看长线基本面，只看情绪接力、资金打板与恐慌冰点抄底。
- **渴望克服人性弱点的散户**：需要一个“冷血无情”、能绝对理性地指出风险点和设定止盈止损位的独立交易系统。

## 三、 产品核心功能矩阵

### 1. 🚀 多头与空头雷达 (涨跌榜异动狙击)
系统自动穿透新浪财经底层接口，瞬时拉取全市场表现最极端（涨停/跌停）的股票数据。
- 引入 **资金势能仪表盘** 与 **柱状图分布**，在毫秒级将枯燥的数据降维成极具视觉冲击力的图表。
- **AI 定向扫雷**：结合实时新闻，判断暴跌的真实原因是主力“恶意洗盘”还是基本面“彻底爆雷”。

### 2. 🔍 FAISS 向量深搜 RAG (避免幻觉)
与传统的文字拼接不同，本产品集成完整的企业级 RAG 架构。
- 将东方财富长篇研报切片、清洗，并注入智谱 `embedding-3` 高维语义库。
- 用户发起个股问询时，执行毫秒级 L2 暴力检索，过滤噪声，将最接近意图的片段提取为上下文，确保 AI 分析的“证据确凿”。

### 3. 🤔 自主行动推理 Agent
引入了类 `AutoGPT` 的 ReAct 模式。只需用户一句话，Agent 将：自主决定调用 `行情探针` 工具 -> 获取数据 -> 自主决定调用 `RAG 检索` 工具 -> 对齐舆情 -> 最终给出研报。

### 4. 💼 沙盒博弈：全拟真持仓系统
包含闭环的虚拟买卖链路：个股买入、现价追踪、实时动态盈亏清算、清仓对撞。结合 Streamlit `Session State` 设计，保证状态无损长驻，提供安全的“试错演兵场”。

## 四、 竞品对比与核心优势

| 对比维度 | 传统炒股软件 (如某花顺) | 常规 ChatGPT/Kimi | **A_Share_Agent (本项目)** |
| :--- | :--- | :--- | :--- |
| **实时性** | 极强 | 弱（依靠旧训练数据或被动搜索）| **强（AKShare 毫秒级直连）** |
| **个性化推演** | 弱（千篇一律的指标） | 强（但极容易发生幻觉） | **极强（RAG 增强 + 游资定制 Prompt）**|
| **自主决策力(Agent)**| 无 | 无 | **有（Function Calling 工具循环）** |
| **系统侵入感** | 重度客户端 | 网页版 | **轻量级全栈流式 Web（响应快）**|

## 五、 产品未来迭代演进 (V3 规划)
1. 接入更加深度的 Level-2 超大单资金流入流出监测（目前仅为 Level-1）。
2. 在模拟推盘的基础上，加入多用户协同（SQLite化）。
3. 允许用户自定义“投资性格”（温和/激进/极致量化），并动态调整 Agent 的 Temperature。
